unit tests for liblisp interpreter public interface

Todo:

It is possible to test if a function throws an assertion when passed invalid data. If it does not throw an assertion it might irrevocably corrupt the program state.

This unit test framework should aim for 100% coverage in each file, however difficult it might be, Gcov can be used to get the current coverage percentage: (see https://gcc.gnu.org/onlinedocs/gcc/Gcov.html).

Note
This could be moved to "util.c" so it can be reused, the unit test functionality has been encapsulated in a series of functions here that should be quite module, if limited to single threaded applications.
All functions under test should have assertions turned on when they were compiled. This test suite can handle SIGABRT signals being generated, it will fail the unit that caused it and continue processing the next tests.
